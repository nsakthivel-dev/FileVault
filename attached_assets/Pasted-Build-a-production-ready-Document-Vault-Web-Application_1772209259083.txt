Build a production-ready Document Vault Web Application where authenticated users can securely:

Upload documents

View uploaded documents

Download documents

Preview supported file types

The application must use:

Frontend: Next.js (App Router) + TypeScript

Backend: Supabase (Auth + Database + Storage)

UI: TailwindCSS + shadcn/ui (or equivalent high-end component system)

Database & Storage: Supabase PostgreSQL + Supabase Storage

The design must look like a premium SaaS product, not a default developer template.

1️⃣ Authentication (Supabase Auth)

Use Supabase Auth with:

Email + password authentication

Secure session handling

Server-side session validation

Protected dashboard routes

Users must only access their own documents.

2️⃣ Database Design (Supabase PostgreSQL)

Create the following table:

documents
Column	Type
id	uuid (primary key, default uuid_generate_v4())
user_id	uuid (foreign key → auth.users.id, on delete cascade)
original_name	text
stored_path	text
file_size	bigint
mime_type	text
created_at	timestamp with time zone (default now())
Row Level Security (MANDATORY)

Enable RLS and create policies:

Users can only SELECT their own documents

Users can only INSERT their own documents

Users can only DELETE their own documents

No public access allowed.

3️⃣ File Storage (Supabase Storage)

Create a private bucket:

document-vault

Rules:

Files stored as:

{user_id}/{uuid}.{extension}

Max file size: 10MB

Allowed types:

application/pdf

image/png

image/jpeg

text/plain

application/vnd.openxmlformats-officedocument.wordprocessingml.document

Files must not be publicly accessible.
Use signed URLs for viewing/downloading.
